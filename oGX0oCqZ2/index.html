Vercel ＋ Github仓库全自动部署Hexo

Author: 温故而知新 Date: 2021-02-22 Reading Time:4.2 mins words:989 Category: hexo note

前言

之前博客系统选择了Gridea，只是因为比Hexo好部署，不需要本地安装一堆环境，而且带windows客户端，但是最近找到了Hexo很方便的全自动部署方法了。

用到的工具

  • Github 静态博客程序存放仓库
  • Vercel 自动部署 + 网站托管 ＋CDN，可以绑定域名
  • 静态博客编辑器 在线Markdown编辑器，支持Github仓库文件管理，可以拿来写更新博客

Vercel一键部署Hexo

  1. 点击这个按钮，一键部署，也可以使用官方版本部署。
  2. 输入项目名称，好像不能用大写。
  1. 创建新仓库，推荐勾选private权限。
  1. 部署项目

部署完成后，会自动分配一个二级域名。你想使用 Vercel 提供的二级域名，直接添加 xxx.now.sh(已经不能添加) 或 xxx.vercel.app ，如果无人占用，会自动验证成功，如何添加下面会写。

项目已经正常运行

  1. 之后，进入控制台

  1. 依次进入setting–> git–> Ignored Build Step 填上下面代码,然后保存。

    git diff --quiet HEAD^ HEAD README.md
  1. 要绑定域名的话，要先添加DNS域名记录，Vercel自带全局 CDN，需要关闭其他CDN。
Type Name Value
A @ 76.76.21.21

如果是子域名，请在你的 DNS 解析处添加 CNAME 记录，指向 cname.vercel-dns.com。

  1. 进入setting–>Domains，绑定自己的域名
  1. 部署完毕

至此 Vercel 的部署过程就结束了。用绑定的域名试一下就可以访问了。

点击这里可以查看运行日志。

因为是把别人Hexo仓库直接fork过来了，所以需要进行修改。

修改网站设置

GitHub仓库文件修改方式很多，这里直接用GitHub的文件管理系统修改。

  1. 更换主题

默认的主题刚好是喜欢的hexo-theme-butterfly，微调一下就好。

  1. 修改网站信息

相关文档可以查看作者博客

2.1 修改根目录下_config.yml文件

# 设置网站标题
title: 温故而知新
# 副标题，可不设置
subtitle: ''
# 修改网站描述
description: 'The universe has to move forward. Pain and loss, they define us as much as happiness or love. <br> 宇宙必须向前发展。痛苦与失去和幸福与爱一样定义了我们。'
# 关键字，多个需要用,分开
keywords:
# 设置作者
author: Bruitiste
#语言设置，中文为zh-CN
language: zh-CN
#设置时区，
timezone: Asia/Shanghai

# URL
## If your site is put in a subdirectory, set url as 'http://example.com/child' and root as '/child/'
# 更新域名
url: https://moeo.info

2.2 修改/themes/butterfly/_config.yml文件

#修改顶部导航
menu:
  主页: / || fas fa-home
  # Home: / || fas fa-home
  #编辑器入口: https://jingtaiboke.com/ || fas fa-file-code-o
  所有文章: /archives/ || fas fa-archive
  标签: /tags/ || fas fa-tags
  #Categories: /categories/ || fas fa-folder-open
  分类:  /categories/  ||  fa  fa-folder-open
  # List||fas fa-list:
  #   - Music || /music/ || fas fa-music
  #   - Movie || /movies/ || fas fa-video
  # Link: /link/ || fas fa-link
  # About: /about/ || fas fa-heart

#设置logo和头像，文件位于/themes/butterfly/source/img之下
# Favicon（網站圖標）
favicon: /img/logo(4).png

# Avatar (頭像)
avatar:
  img: /img/avatar.jpg
  effect: false

#修改评论系统
comments:
  # Up to two comments system, the first will be shown as default
  # Choose: Disqus/Disqusjs/Livere/Gitalk/Valine/Waline/Utterances/Facebook Comments/Twikoo
  use:
  - Valine
  # - Disqus
  text: true # Display the comment name next to the button
  # lazyload: The comment system will be load when comment element enters the browser's viewport.
  # If you set it to true, the comment count will be invalid
  lazyload: false
  count: true # Display comment count in top_img
  
#可以设置一下页脚，插入网站运行时间
# Footer Settings
# --------------------------------------
footer:
  owner:
    enable: true
    since: 2021
  custom_text:
  copyright: false # Copyright of theme and framework

详细设定，可以参考主题作者博客。

  1. 清空旧文章

清空/source/_posts/下文件即可。

在线更新博客

可以使用静态博客编辑器管理，也可以本地写好，贴到GitHub上。

在/source/_posts/添加新文章，即可自动生成新网页，新文件格式可以参考/scaffolds目录下，文件。
